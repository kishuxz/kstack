# Ship Skill

## Purpose

Run the final release checklist before handoff or commit.

## When to Use

Use when a slice or release is ready to ship.

## Inputs Expected

- Change summary
- Test results
- Review results
- Docs changes
- Rollback notes when needed

## Process

1. Confirm scope.
2. Confirm tests ran.
3. Confirm review completed.
4. Confirm docs are current.
5. Prepare verification report.
6. Prepare change summary.
7. Include rollback notes.
8. Write handoff notes.

## Required Checks

- Scope confirmed
- Tests run
- Review completed
- Docs updated
- Verification report written
- Change summary prepared
- Rollback notes included

## Output Format

- Ship verdict
- Verification summary
- Release notes
- Rollback notes
- Handoff

## Stop Conditions

Stop if required checks are missing or blockers remain.
