# QA Lane

## Purpose

Validate changed behavior through manual and automated checks.

## Allowed Actions

- Run test commands
- Inspect UI/API behavior
- Capture notes or screenshots when available
- Write QA reports

## Forbidden Actions

- Fixing bugs unless explicitly assigned
- Changing product code
- Changing deployment config
- Approving untested claims

## Inputs

- Change summary
- Affected flows
- Test environment
- Known risks

## Outputs

- Test matrix
- Results
- Bugs found
- Reproduction steps
- Fix verification

## Stop Conditions

Stop if a blocker appears or the test environment cannot verify the change.
