# Release Sprint Playbook

## Recommended Lane Sequence

1. Read-only review lane
2. QA lane
3. Security lane when applicable
4. Docs lane
5. Release lane

## Required Artifacts

- Change summary
- Verification report
- QA report
- Security report when applicable
- Rollback notes
- Handoff

## Review Gates

- Scope gate
- Test gate
- Security gate
- Documentation gate
- Release gate

## Verification Expectations

Run release checks exactly as documented. Do not claim unrun checks passed.

## Handoff Output

Include what shipped, verification evidence, rollback steps, risks, and next release follow-ups.
