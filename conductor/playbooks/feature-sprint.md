# Feature Sprint Playbook

## Recommended Lane Sequence

1. Spec
2. Plan
3. Implementation lane
4. QA lane
5. Read-only review lane
6. Docs lane
7. Release lane

## Required Artifacts

- Spec
- Implementation plan
- Test plan
- Verification report
- Change summary

## Review Gates

- Scope gate
- Architecture gate
- Test gate
- Security gate when sensitive areas changed
- Documentation gate

## Verification Expectations

Run relevant automated tests and manual QA for affected flows.

## Handoff Output

Summarize shipped behavior, verification, risks, and next work.
