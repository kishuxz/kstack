# Refactor Sprint Playbook

## Recommended Lane Sequence

1. Spec
2. Plan
3. Read-only review lane
4. Implementation lane
5. QA lane
6. Release lane

## Required Artifacts

- Refactor goal
- Non-goals
- Allowed and forbidden files
- Test plan
- Rollback notes

## Review Gates

- Scope gate
- Architecture gate
- Behavior preservation gate
- Test gate

## Verification Expectations

Run tests that prove behavior stayed the same unless behavior changes were explicitly approved.

## Handoff Output

Summarize structural changes, unchanged behavior, verification, and follow-up risks.
